开篇

Posted by Qiang Lv on 2017-11-02

导言

微服务最近真的很火，每个搞架构的都希望自己的业务转向微服务。微服务架构与传统架构模式相比，具有语言无关性、独立进程通讯、高度解耦、任务边界固定、按需扩展等特点，适合互联网公司快速交付、响应变化、不断试错的需求。

为什么是Spring Cloud

Spring Cloud几乎覆盖了微服务架构的方方面面，包括了配置管理、服务发现、断路器、智能路由、微代理、控制总线、全局锁、决策竞选、分布式会话和集群状态管理等内容。虽然2016年才出来Releasev1.0版本，但是已经大量用在各大互联网公司。从目前的关注度和活跃度来看，Spring Cloud很可能会成为微服务架构的标准。

为什么要记录

从开始接触Spring Cloud已经过了将近一年的时间，大大小小练习了几个项目，但是让我脱稿说出Spring Cloud框架内容和核心代码内容，还是只能吞吞吐吐不知所言。因此计划这次尽量花
一段时间来记录从0到1的项目研发。从表面到底层都能记录下所做工作。年纪大了脑子真是不好用了。
