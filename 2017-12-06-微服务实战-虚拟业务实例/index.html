微服务实战-虚拟业务实例

Posted by Qiang Lv on 2017-12-06

一些废话

文章隔了一个多月才开始更新，期间出了两次国潇洒了二十几天，回来后抽时间把项目架构和一些简单的模块完成，参照代码把这次项目记录下来。

虚拟业务简介

在我的认知中微服务更适合快速业务试错的模式，在项目开始到最终项目上线，spring cloud微服务框架可以节约开发人员大量的时间。所以在挑选这次实战业务项目上我选择了我熟悉的和o2o项目，在13年开始大量未知名的o2o项目如雨后春笋冒了出来，到17年存活的项目已经不多了。凑巧一天在地铁上看到以前公司老大分享了一篇连咖啡的文章，介绍了连咖啡的一些产品，就好奇找到公众账号看了看，其中产品很简单，开发起来用Spring cloud也非常合适，因此就好巧不巧就决定使用连咖啡的产品模式作为这次的项目虚拟业务实例。如果不了解连咖啡的可以找到公众账号了解其产品。

涉及到技术

本次项目为了基于快速上手、快速研发、快速部署这三个特点，使用了以下技术：

  • Spring Boot
  • Spring Cloud
  • Spring Security
  • H2
  • Docker

使用机器：

  • MacBook Pro (Retina, 13-inch, Early 2015)
  • MacOS High Sierra
  • 3.1 GHz Intel Core i7
  • 16 GB 1867 MHz DDR3

由于本人对于前端css、js技术不太熟悉，所以本次项目不涉及到前端界面研发。
